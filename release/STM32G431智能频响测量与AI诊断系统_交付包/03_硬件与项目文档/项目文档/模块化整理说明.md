# STM32G431 频响测量系统模块化整理说明

> 历史整理说明：当前正式主线以 `项目总览与主流程.md` 为准。本文件保留早期模块化记录，适合作为竞赛文档素材，不作为当前软件入口说明。

本文用于支撑嵌入式芯片与系统设计竞赛文档中的“软件架构”和“模块划分”章节。

## 一、系统总体链路

本项目由 STM32G431 固件和 Python 上位机组成一条完整测量链路：

1. 固件通过 DAC1/PA4 输出正弦激励。
2. TIM2 触发 DAC DMA 循环输出，TIM1 触发 ADC1/ADC2 同步采样。
3. ADC1/PA0 作为输入参考通道，ADC2/PA1 作为输出响应通道。
4. 固件对每个频点做 FFT 幅相提取，得到传递函数 `H(jw)=ADC2/ADC1`。
5. UART 输出三组兼容数组：
   `omega=[...]`、`Magnitude_data=[...]`、`Phase_data_rad=[...]`。
6. 上位机读取三数组和可选诊断数组，完成 Bode、Nyquist、滤波器类型、阶次、截止频率和稳定性分析。

## 二、上位机模块划分

上位机已按职责拆分为以下模块：

| 文件 | 职责 |
| --- | --- |
| `系统分析/main.py` | Tkinter GUI 入口、参数输入、文件导入导出、绘图与日志显示。保留 `analyze_system` 等兼容导出，旧测试和脚本仍可从 `main` 导入。 |
| `系统分析/filter_analysis.py` | 纯算法层：数据质量筛选、平滑、滤波器类型识别、阶次估计、截止频率、Nyquist 稳定性、报告文本生成。 |
| `系统分析/serial_protocol.py` | 串口文本协议层：三数组解析、诊断数组解析、SWEEP 命令生成、串口帧读取。 |
| `系统分析/serial_readers.py` | 后台线程层：单帧读取、连续读取、自动识别补扫，负责串口读写流程但不直接绘图。 |
| `系统分析/serial_transport.py` | 串口传输适配层：打开串口、发送 ASCII 命令。 |
| `系统分析/measurement_model.py` | 数据模型层：`MeasurementFrame` 保存一次完整测量帧及诊断信息。 |

这种划分让 GUI、通信协议、数据模型和分析算法解耦，后续可以单独测试算法，也可以替换串口传输方式而不影响图形界面。

## 三、固件模块划分

固件仍保持 STM32CubeMX 生成工程结构，避免破坏 Keil/MDK 工程配置。主要自定义逻辑集中在 `G431CBTx_sweep/Core/Src/main.c` 的 `USER CODE` 区，按函数职责整理如下：

| 模块 | 代表函数/数据 | 职责 |
| --- | --- | --- |
| 扫频配置 | `StepSweep_SetDefaultConfig`、`StepSweep_ValidateConfig` | 设置默认扫频范围、幅值、步进并校验输入参数。 |
| UART 命令协议 | `SweepProtocol_ReadLine`、`SweepProtocol_ParseCommand`、`SweepCommand_Execute` | 支持 `HELP`、`PING`、`DEFAULT`、`SWEEP` 命令。 |
| 定时器与采样调度 | `StepSweep_ConfigureAdcTimer`、`StepSweep_ApplyTimerSettings`、`StepSweep_SettleTimeMs`、`StepSweep_CaptureTimeoutMs` | 按当前频点配置 TIM1/TIM2、计算稳定等待时间和 ADC 捕获超时。 |
| 正弦激励生成 | `StepSweep_Generate_Data` | 按实际 DAC 缓冲长度和周期数生成相干正弦波。 |
| 幅相分析 | `wave_analys`、`unwrap_phase` | 去直流、FFT、计算幅值比和相位差，并做相位解缠。 |
| 结果与诊断输出 | `StepSweep_ClearPointResult`、`SweepOutputSummary_*`、`UART_Print*Array` | 保存每个频点的幅值、相位、RMS、DC、削顶标志，并输出兼容三数组和诊断数组。 |

关键兼容性保持不变：固件仍先输出 `omega`，再输出 `Magnitude_data`，最后输出 `Phase_data_rad`。这些名称和顺序是上位机与 MATLAB 文本导入的兼容边界。

## 四、后续建议

如果后续要继续深入模块化固件，可以新增 `sweep_engine.c/.h` 与 `sweep_protocol.c/.h`，再同步更新 Keil `.uvprojx`。当前阶段优先保持工程可打开、功能不变、协议不变，因此只做了 `main.c` 内部函数级整理。

本次验证已完成 Python 语法检查和上位机单元测试。当前环境未发现 Keil/ARM 命令行编译器，因此固件部分需在 Keil MDK 中再执行一次工程编译确认。
