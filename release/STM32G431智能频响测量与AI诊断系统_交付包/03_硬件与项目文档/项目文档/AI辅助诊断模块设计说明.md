# AI辅助电路识别与智能诊断模块设计说明 v2

> 历史设计说明：当前正式主线以 `项目总览与主流程.md` 为准。本文件保留 AI 诊断 v2 的实现细节，部分 GUI 描述可能早于当前收口后的主界面。

## 一、系统定位升级

本项目在原有“基于 STM32G431 的频率响应测量仪”基础上，新增并升级 `系统分析/ai_diagnosis.py`、`系统分析/smart_resweep_reader.py` 与 `系统分析/diagnosis_knowledge_base.json`，将作品定位提升为：

**基于 STM32G431 的智能频响测量与 AI 辅助电路诊断系统**。

升级后系统仍保持原 STM32 固件和串口三数组协议不变：

- `omega=[...]`
- `Magnitude_data=[...]`
- `Phase_data_rad=[...]`

上位机继续兼容旧文本导入、Bode 图、Nyquist 图、CSV 保存和原有自动识别流程。AI 诊断 v2 只在 Python 侧增加智能分析、故障诊断和主动补扫能力。

当前实现采用“传统频响分析为主判据，AI 模块为增强诊断层”的结构。也就是说，`filter_analysis.py` 中已经经过标准数据验证的滤波器类型和阶次判断仍然负责最终拍板；当该传统判据置信度高、阶次可靠时，AI 候选排序必须服从该结果。AI 负责补充工程师更关心的内容：等效传递函数拟合、候选证据链、测量质量诊断、故障知识库判断和下一步补扫建议。

这样设计的原因是：竞赛仪器首先要“判得准”，其次才是“看起来智能”。AI 不应该推翻已经更稳定的物理判据，而应该把原有判断解释清楚，并在数据质量差、扫频范围不足或候选接近时给出复测策略。

## 二、为什么采用轻量特征模型而不是大模型

本模块没有直接采用深度学习大模型，而是采用“频响特征提取 + 传递函数物理模板拟合 + 多候选置信度 + 故障知识库”的轻量可解释方案，主要原因如下：

1. 低通、高通、带通、带阻响应具有明确的电路物理规律，幅频斜率、-3 dB 交点、相位跨度和峰谷形态本身就是强特征。
2. 竞赛现场更看重可复现、可解释、可验证。轻量模型可以说明“为什么判为二阶低通”，而不是给出黑盒标签。
3. 当前八类标准/实测样本数量有限，直接训练神经网络容易过拟合，且难以解释误判原因。
4. 轻量模型可以在普通 Windows 上位机、树莓派或离线环境稳定运行，不依赖 GPU、网络和云服务。
5. 后续如果积累更多数据，可以在当前特征层之上加入朴素贝叶斯、决策树或小型机器学习分类器，不破坏接口。

因此这里的“AI”不是噱头，而是把工程师根据频响曲线识别电路、发现异常、安排下一轮测试的经验固化为可计算、可解释、可回归测试的流程。

## 三、模块输入与输出

主入口保持向后兼容：

```python
run_intelligent_diagnosis(
    omega,
    magnitude,
    phase_rad,
    diagnostics=None,
    analysis_result=None,
)
```

输入包括：

- 角频率数组 `omega`
- 幅值数组 `Magnitude_data`
- 相位数组 `Phase_data_rad`
- 可选诊断数组 `diagnostics`，例如 `Input_rms_v`、`Output_rms_v`、`Input_dc_v`、`Output_dc_v`、`Clip_flags`
- 可选传统分析结果 `SystemAnalysisResult`

`IntelligentDiagnosis` 保留原字段，同时新增 v2 字段：

- `candidates`：多候选排名，例如二阶低通、一阶低通、三阶低通
- `best_fit`：当前主候选对应的物理模板拟合
- `model_fits`：所有模板拟合结果
- `fault_findings`：由故障知识库触发的证据链
- `active_sweep_plan`：可一键执行的 1 到 3 条 `SWEEP` 补扫计划
- `needs_resweep_confirmation`：当第一、第二候选置信度接近时提示需要补扫确认
- `decision_strategy`：说明本次诊断是“传统判据锚定”还是“特征/模板融合判断”

## 四、特征提取方法

`extract_diagnosis_features()` 从幅频、相频和诊断数组中提取关键工程特征：

- 低频增益 `low_gain_db`
- 高频增益 `high_gain_db`
- 最大/最小幅值 `max_gain_db`、`min_gain_db`
- 通带/阻带差值 `pass_stop_span_db`
- 峰值余量 `peak_margin_db`
- 陷波深度 `notch_margin_db`
- 低频上升斜率 `head_slope_db_dec`
- 高频滚降斜率 `tail_slope_db_dec`
- 相位跨度 `phase_span_deg`
- -3 dB 交点数量 `minus3_crossing_count`
- Nyquist 曲线到 `(-1, 0)` 的最小距离
- 输入/输出 RMS 中位数
- 输入/输出 DC 偏置
- PA0/PA1 削顶点数量
- 最少有效采集次数
- 扫频覆盖 decade 数

这些特征会进入候选评分和故障规则，并在报告底部输出“关键特征”，便于比赛现场解释判断依据。

## 五、传递函数模板拟合

AI 诊断 v2 新增 `fit_transfer_templates()`。它使用 dB 幅频进行物理模板拟合，相位作为辅助证据，不作为强约束。

模板包括：

- 低通：一阶、二阶、三阶
- 高通：一阶、二阶、三阶
- 带通：二阶中心频率、带宽、Q 值
- 带阻：二阶陷波中心、带宽、Q 值，并允许陷波底部受噪声底限制

有 `scipy.optimize` 时优先使用 `least_squares`。如果现场电脑没有安装 SciPy，模块会自动使用轻量网格拟合后备算法，仍能输出 `TransferModelFit`、R2、RMSE 和参数说明。

每个拟合结果包含：

- `model_family`
- `order`
- `parameters`
- `r_squared`
- `rmse_db`
- `residual_summary`
- `evidence`

报告示例：

```text
模型拟合: 低通二阶模板: fc=1021.8 Hz, R2=0.998；RMSE=0.94 dB, 最大残差=2.1 dB；参数={gain_db=-0.02, fc_hz=1.02e+03}
```

## 六、多候选置信度

`rank_diagnosis_candidates()` 综合三类证据：

1. 特征规则分：低频/高频增益、峰谷位置、斜率、相位跨度等。
2. 模板拟合分：R2 越高、RMSE 越低，物理模板可信度越高。
3. 传统分析分：复用 `filter_analysis.py` 已验证的识别结果。

最终报告输出 Top 3 候选，例如：

```text
候选排名: 二阶低通系统 0.99 / 三阶低通系统 0.86 / 一阶低通系统 0.81
```

主结果取最高分候选。当最高分和第二名差距小于 0.12 时，报告会标记“需补扫确认”，并优先建议执行智能补扫。

## 七、独立故障知识库

故障规则独立放在：

```text
系统分析/diagnosis_knowledge_base.json
```

使用 Python 标准库 JSON 读取，不引入 YAML 等额外依赖。当前知识库包含：

- PA0 输入参考削顶
- PA1 输出响应削顶
- PA1 偏置接近 ADC 边界
- 高通/带通输出偏置异常
- 输入 RMS 过低
- 输出接近噪声底
- PA0/PA1 疑似接反或有源增益异常
- 扫频范围不足
- 有效采集次数不足
- 带阻陷波深度受噪声底限制
- 高频寄生影响疑似存在

每条规则输出：

- 故障名
- 严重度
- 证据文本
- 建议文本
- 是否扣减置信度

报告示例：

```text
故障证据链: PA1输出响应削顶(critical): PA1削顶频点 4 个...
可能故障原因: PA1输出响应削顶: 降低激励幅值，检查运放供电范围、输出摆幅和PA1输出节点偏置。
```

## 八、主动补扫策略

`build_active_sweep_plan()` 根据识别结果生成可执行补扫命令：

- 低通/高通：围绕截止频率加密扫频。
- 带通/带阻：围绕左右 -3 dB 边界分别加密扫频。
- 未知响应：扩大扫频范围并降低步进。

命令仍使用现有固件格式：

```text
SWEEP <start_hz> <stop_hz> <step_hz> <amplitude_vpp>
```

例如：

```text
一键补扫计划: SWEEP 615 2560 100 0.6 (围绕低通截止频率 1022 Hz 加密扫频)
```

## 九、GUI 集成方式

`系统分析/main.py` 集成方式：

- 每次 `handle_frame()` 完成传统分析后，自动调用 `run_intelligent_diagnosis()` 并输出报告。
- 新增“智能诊断”按钮，可对当前帧重新生成报告。
- 新增“智能补扫”按钮，读取 `active_sweep_plan` 并自动发送 1 到 3 条 `SWEEP` 命令。
- 补扫线程由 `smart_resweep_reader.py` 实现，复用现有串口读取逻辑，采集后调用 `merge_measurement_frames()` 合并原始帧和补扫帧，再重新进入传统分析与 AI v2 诊断。
- 停止按钮仍然有效；停止事件触发后不会继续发送剩余补扫命令。

GUI 不新增固件命令，不改变 UART 输出数组名和顺序。

## 十一、创新性提升

AI 诊断 v2 让系统从“能测频响”升级为“能自动判断、自动诊断、自动建议下一步测试”的智能仪器：

1. 自动识别低通、高通、带通、带阻和未知响应。
2. 自动估计阶次、截止频率、中心频率和带宽。
3. 用物理模板拟合给出 R2、RMSE 和参数证据。
4. 用多候选置信度展示不确定性，而不是只给单一标签。
5. 用故障知识库输出可追溯的测量质量诊断。
6. 用一键主动补扫把诊断结果转化为下一轮测量动作。
7. 保持 STM32G431 固件和串口协议不变，工程实现风险低，展示时可解释性强。

这使作品更接近真实工程仪器：不仅画出曲线，还能解释曲线、诊断异常、指导操作者完成下一轮更可靠的测量。
